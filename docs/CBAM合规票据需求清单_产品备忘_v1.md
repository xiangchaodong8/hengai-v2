## 1. 用户痛点

合规专员知道要「升格为可核验结果」，但**手上没有原始票据**，也无法一次性说清该找财务/采购要什么。  
系统应主动输出一份 **可转发同事的《票据需求清单》**，而不是只给抽象指引。

**产品目标**：把「我不知道要什么材料」变成「复制清单 → @财务 @采购 → 按单索骥」。

---

## 2. 标准输出结构（对话已验证的模板）

### 2.1 清单头

| 字段 | 示例 |
|------|------|
| 标题 | 《CBAM 升格核算 · 票据清单》 |
| 企业 | 比亚迪汽车股份有限公司 |
| 对应 HS | 7604.10 |
| 出口国 | 德国、法国 |
| 生成依据 | 当前 AppState 测算上下文（产品、默认值、Scope3 覆盖率缺口） |

### 2.2 分部门条目（每条固定四行）

```
第一类：{部门名}提供   ← 如「财务」「采购」

文件：{具体文件名/类型}
关键数据：{可量化字段}
用途：{替换哪个默认值 / 填补哪块 Scope3}
备注：{分厂电表、供应商暂无碳数据时的降级策略等}
```

**对话样例（铝出口场景）**：

- **财务**：年度电费结算单 / PPA → 各月 kWh + 峰谷平 → 替换默认 3,000,000 kWh，匹配 east 电网因子  
- **采购**：铝锭合同 + 磅单/质证书 → 供应商 + 吨数 + 铝液碳强度（若有）→ 填补 Scope3 覆盖率 0  

### 2.3 收尾 CTA（须对齐全域中心架构）

```
拿到上述文件后，进入 [全域中心] → [CBAM 测算工具]，
在 Step 4 点击「升格为可核验结果」上传即可。
系统将匹配因子、扣除模拟漂移，生成带 certId 的可申报凭证。
```

**禁令**：不得指引「右侧面板上传」等虚构方位；上传入口以 `[企业数字档案]` / CBAM Step 4 升格为准。

## 3. 触发场景

| 场景 | 触发方式 |
|------|----------|
| 用户说「没有票据 / 没有电费单 / 供应商不配合」 | 对话 AI 直接输出清单（**当前已可通过 prompt 做到**） |
| CBAM Step 4 · `SIMULATED` 态 | Evidence Bar 次要入口：「生成发给同事的票据清单 →」 |
| 升格前拦截 | `PENDING_VERIFICATION` 前检查关键字段缺失 → 先生成清单再引导上传 |
| 供应链沉默 | 与「需申请因子」联动：清单中增加「向原厂申请因子授权」条目 |

## 5. 与双模状态机关系

| evidence.mode | 清单角色 |
|---------------|----------|
| `SIMULATED` | 主交付物：告诉用户「要升格，先凑这些」 |
| `PENDING_VERIFICATION` | 补缺清单：已上传项打勾，剩余项高亮 |
| `SOVEREIGN_VERIFIED` | 可选「年度更新清单」— 维护 drift 用 |

---

## 6. 开发纪律

- 清单内容必须 **AppState 驱动**（企业名、HS、默认 kWh、Scope3 覆盖率等），禁止静态 demo 假数据
- 首页对话可输出 Markdown 清单；**重型导出/PDF 走 Hub 或 Widget**，不撑爆聊天气泡
- 不提前实现未排期的 PDF/邮件网关；Phase 1 以 **复制粘贴 + 全域中心升格路径** 闭环即可
