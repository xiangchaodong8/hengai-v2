# 精算芯 · 实证兑换与导出权限契约 v1

> **状态**：产品/架构锁定 · **Hub redeem 主路径已实现**（2026-06-23 供应链批次）  
> **前提**：精算芯为 **独立付费工具插件**；Hub 不算力、不存工序绝对值。  
> **关联**：`docs/SYNC_CONTRACT_v2.md` · `docs/三界定位_v2.md` · `docs/双模状态机_开发契约_v1.md`  
> **原则**：**不追求智能静默并网**；以 **钥匙码 / 二维码兑换入库** 为产品主路径；**并非所有精算芯使用者均可导出或乱用结果**。

---

## 1. 设计立场（用户确认）

| 维度 | 要求 |
|------|------|
| **严肃** | 升格/确权 = 显式「持钥入库」，可审计、可交代 |
| **安全** | 精算芯本地 vault 为真理源；Hub 只收 **验签后的 L0/L1 摘要** |
| **专业** | 付费边界、企业绑定、角色卡口；禁止「谁都能导出、谁都能兑」 |
| **非目标** | Hub 自动拉 :8001、共库、iframe 套娃、聊天里静默改状态 |

## 2. 两条入库通道（职责分离）

| 通道 | 面向 | 入口 | 权限模型 |
|------|------|------|----------|
| **A · 兑换入库（产品主路径）** | 付费精算芯用户 | Hub「导入精算芯实证包」填 **钥匙码** / 扫 **QR** | §4 导出权 + §5 兑换权 |
| **B · overview/sync（工程联调）** | 开发 / e2e | `POST /api/v1/hub/overview/sync` | 同 JWT + `productionEntity` 绑定；**不替代** A 的产品权限 |

**纪律**：生产环境默认 **只开放 A**；B 仅内网 / staging / `e2e_hub_sync.py`。

## 3. 精算芯侧：谁可以「产出」兑换物

并非「用过精算芯」即可生成可流通的钥匙码/QR。

### 3.1 许可证层级（Core 本地校验）

| 层级 | 可本地测算 | 可生成 CL-GTCID 草稿 | 可 **签发兑换包**（钥匙码/QR） |
|------|------------|----------------------|--------------------------------|
| **试用 / 未付费** | 受限次数或样例数据 | 否 | **否** |
| **已订阅（个人/企业席）** | 是 | 是（本企业） | **是**，且绑定 `productionEntity` |
| **席位过期 / 吊销** | 只读历史（可选） | 否 | **否**；已发码按 §5.4 作废策略 |

### 3.2 导出物类型（分级，防乱用）

| 类型 | 内容 | 谁可生成 | 可否被 Hub 外第三方读取 |
|------|------|----------|------------------------|
| **E0 · 本地报告 PDF** | 全工序复盘整本（仅本机） | 付费席 + 企业管理员角色 | **否**（不出 Core 签名流通包） |
| **E1 · 兑换包（钥匙码/QR）** | L0/L1 脱敏摘要 + `certId` + `encHash` + 签名 | 付费席 + **签发员**角色 | 仅持有码者可向 **自家 Hub** 兑换一次 |
| **E2 · 公域 Pull 因子** | 正式城池 certified 后 | 魔盒封签 + Hub 规则 | 下游 Pull 路径（另契约） |

**禁令**：

- 禁止「一键导出全部 vault JSON」给非管理员  
- 禁止未付费用户生成 **可兑换** QR/钥匙码  
- 禁止兑换包内含 **工序绝对能耗 / processes[]**（对齐 SYNC §1）

### 3.3 钥匙码 / QR 载荷（概念）

```
兑换包 = {
  redeemId,           // 一次性 ID
  certificateId,      // CL-GTCID
  productionEntity,   // 统一社会信用代码 / CL 主体 ID（硬绑定）
  carbonIntensity,    // 摘要
  encHash, dataFingerprint,
  dictVersion, calcVersion,
  maturityTier, matBoxLocked,
  issuedAt, expiresAt,   // 建议 7–30 天
  issuerSeatId,            // 精算芯付费席位
  signature                // CL 私钥 / HMAC（Hub 公钥验签）
}
```

- **钥匙码**：短码 → Hub 后端查表或解析 signed payload（人类录入场景）  
- **QR**：`https://hub…/redeem?t=ONE_TIME_TOKEN` 或 embedded signed blob（扫码场景）  
- **一码一_workspace_一次**；重复兑换返回 `ALREADY_REDEEMED`

## 4. Hub 侧：谁可以「兑换入库」

### 4.1 兑换前置条件（全部满足）

1. Hub **已登录**，且账号属于目标 **Workspace**  
2. Workspace 的 `creditCode` / `productionEntity` **与兑换包一致**（防串号、防乱用）  
3. 兑换包 **签名有效、未过期、未使用**  
4. 兑换人角色 ≥ **企业合规官 / 管理员**（普通访客 **不可** 替企业兑入）  
5. `issuerSeatId` 在 CL 订阅系统中 **状态=active**（吊销则拒兑）

### 4.2 兑换后状态（与双模状态机一致）

| 包内 `maturityTier` / `matBoxLocked` | Hub `cityState` | `EvidenceMode` | Pull |
|--------------------------------------|-----------------|----------------|------|
| L1 软件实证 | `evidence_building` | `PENDING_VERIFICATION` | 否 |
| L2/L3 + 未封签 | `mat_pending` | `PENDING_VERIFICATION` | 否 |
| L3 + `matBoxLocked` | `certified` | `SOVEREIGN_VERIFIED` | 是（另检 Pull 规则） |

### 4.3 谁可以「读」Hub 上的精算结果

| 读者 | 可见范围 |
|------|----------|
| **本 Workspace 成员（合规/管理员）** | 本企业已兑换摘要 + certId + 城池态 |
| **绑定链下游（匿名默认）** | 仅行业大盘 / 「实证中」卡片（§三界定位 下游卡片） |
| **其他企业 / 未绑定用户** | **不可** 读 vault 明细；**不可** 用他人钥匙码兑入自家 Hub |
| **Hub 平台** | 仅存 L0/L1 摘要；**不** 存精算芯本地工序明细 |

## 5. 防滥用规则（核心）

1. **企业绑定**：兑换包 `productionEntity` ≠ 当前 Workspace → `403 ENTITY_MISMATCH`  
2. **席位绑定**：签发席位于 CL 侧吊销 → 未兑换包失效；已兑换不回滚主值（另记 `REVOKED_ISSUER` 风险标）  
3. **一次性**：同一 `redeemId` 二次提交 → `409 ALREADY_REDEEMED`  
4. **速率限制**：同 IP / 同账号 兑换失败 N 次 → 临时锁定（防暴力猜码）  
5. **无批量导出 API**：Hub **不提供**「列出所有精算芯用户结果」接口  
6. **模拟态隔离**：Hub `SIMULATED` 不得被兑换包覆盖为「已确权」除非验签通过且 tier 达标  
7. **Action_Ledger**：`CORE_EVIDENCE_REDEEMED` / `CORE_REDEEM_DENIED` 全量留痕

## 6. 建议 API（排期后实现）

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/v1/hub/evidence/redeem` | body: `{ redeemCode \| qrToken }` → 验签入库 |
| `GET` | `/api/v1/hub/evidence/redeem/history` | 本 Workspace 兑换记录（管理员） |
| *(Core 侧)* | `POST /api/v1/core/evidence/issue-redeem` | **已实现**（`HengAI_Core_Test` · `core_redeem.py`） |

**错误码（Hub redeem）**：

| code | 含义 |
|------|------|
| `INVALID_SIGNATURE` | 伪造或篡改 |
| `EXPIRED_REDEEM` | 过期 |
| `ALREADY_REDEEMED` | 已使用 |
| `ENTITY_MISMATCH` | 非本企业 |
| `SEAT_INACTIVE` | 签发席位无效 |
| `INSUFFICIENT_ROLE` | 兑换人权限不足 |

## 9. 验收要点

1. 未付费精算芯用户 **无法** 生成有效兑换钥匙码/QR  
2. 付费用户 **无法** 将 A 企业签发的码兑入 B 企业 Hub  
3. 非管理员 Hub 账号 **无法** 完成 redeem  
4. 兑换成功后 Ledger 可追溯；重复兑换被拒绝  
5. Hub 上 **无** 精算芯工序绝对值字段
