# HengAI 三界定位 v2

> **版本**：2026-06 · 讨论定稿  
> **适用范围**：精算芯（:8001）· CL-MAT 魔盒 · 全域中心 Hub（:8000）  
> **原则**：Hub 不算力、不存工序绝对值；精算芯可独立付费；魔盒为申报/公证/Pull 主桥；可见性按场景拆分（原厂自证可实名、下游消费默认匿名）。

---

## 1. 三条产品线（勿混改）

| 线 | 工作区 / 端口 | 定位 |
|----|---------------|------|
| **精算芯 · CL-GTCID 算力芯** | `HengAI_Core_Test` · `:8001` / 官网「产业链主权实证中心」 | **工具插件**：多行业工艺审计、因子池、CL-GTCID 权证；可先软件上线，后补硬件实证 |
| **CL-MAT 魔盒** | `cl-mat/` · 盒端 | **商业与物理卡口**：封签、正向捕获、置信 Lv.4；不对 Hub 开发期硬耦合 |
| **全域中心 Hub** | `HengAI-V2` · `:8000` | **指挥台**：城池态机、共振阈值触发、绑定链展示、Pull/核验（仅正式城池） |

纪律：精算芯 **standalone 可运行**；Hub **不迁入** 13 行业精算引擎；官网 OSS **Hub Done 前不切链**（见 `HengAI_Core_Test/docs/全域中心_衔接说明.md`）。

## 2. 精算芯产品边界

### 2.1 可做什么

- 用户（通常为大厂合规员）注册 CL、**付费**使用精算芯
- **反向复盘**真实发生的数据，找碳强度差距
- 产出：行业实证、因子池草稿、**CL-GTCID 权证**（软件层）
- **无魔盒也可下载/线上使用**

### 2.2 不可做什么（无魔盒 / 未 hardware 封签前）

- **不可**用于 CBAM/海关 **申报**
- **不可** **技术公证** 级协同
- **不可**进入 Hub **正式核验池 Pull**（公域因子池）

### 2.3 商业终点

精算芯强调：「算清楚差距」→ 为保险与合规 **妥协 hardware 哈希物理值** → 引导 **安装魔盒**。

## 3. 两种玩法（双漏斗）

### 玩法 A · 精算路径（先软后硬 · 排斥魔盒的大厂）

```
大厂排斥魔盒
  → 自购或下游共振阈值触发开通精算芯
  → 反向实证 + CL-GTCID
  → Hub 展示「实证中」（实名）
  → 中小供应商绑定链 + 行业大盘施压
  → 补装魔盒 · hardware 实证
  → 正式碳城池
```

**Hub 价值**：让下游看到 **「请勿急，我们已在开始了」** — 不是没上游，是 **上游已开工、尚未可 Pull**。

### 玩法 B · 魔盒路径（政策驱动 · 先硬后城）

```
政策 / 合规压力
  → 线下安装魔盒
  → 数据提前捕获
  → Hub「待建城池」预览（实名）
  → 原厂被导向全域中心
  → 正式碳城池
```

**Hub 价值**：安装后即 **待建城池**，比玩法 A 更早、更「实」。

两种玩法 **终点均为正式碳城池**；Hub 展示 **城池态** 不同，下游期待与压力节奏不同。

## 4. 精算芯双入口 + 共振触发（卡口 ①）

| 路径 | 代码建议 | 谁付费 | 谁使用精算 | Hub 信号 |
|------|----------|--------|------------|----------|
| **A1 · 大厂自购** | `fundingMode: self_paid` | 原厂 CL 账号 | 原厂 | `cityState: evidence_building` |
| **A2 · 共振达阈触发** | `fundingMode: resonance_triggered` | 任一供应商可选购 CL-MAT 终端 | 原厂（由阈值触发进入实证） | 共振阈值达标 + 进入实证 |

### 4.1 共振机制（非金融集资）

- **指向**：单一实名 `productionEntity`（CL 主体 / 信用代码）
- **展示**：共振次数、阈值进度、参与企业数（不展示资金归集进度）
- **机制**：
  - 共振次数达阈值后，任一供应商可选择为其购买 CL-MAT 终端
  - 该动作为“终端购买触发实证”，非“第三方资金归集”
- **话术**：「请勿急，我们已在开始了」（自购 = 原厂已开始；共振达阈 = 产业链已触发进入实证）

### 4.2 风险边界（已锁）

- 取消资金归集路径，避免金融法规风险
- 共振仅作为独立 IP 触发机制，不承载第三方集资资金流

## 5. 三层商业卡口

| 卡口 | 产品 | 解锁能力 |
|------|------|----------|
| **① 共振触发** | 中小企业共振达阈（可选） | 触发上游进入实证并可选发起 CL-MAT 终端购买 |
| **② 精算芯** | 付费订阅 / 共振达阈触发 | CL-GTCID、软件实证、Hub **实证中** 展示 |
| **③ 魔盒** | CL-MAT 部署 + hardware 封签 | 待建城池 → 正式城池；Pull / 申报 / 技术公证路径 |

## 6. 碳城池三态（Hub 展示态机）

| cityState | 典型来源 | 下游看到（实名） | Pull / 申报 / 公证 |
|-----------|----------|------------------|---------------------|
| `evidence_building` | 精算芯软件实证、CL-GTCID 已出 | 企业名 + GTCID + 强度摘要 + 「实证中，待魔盒封签」 | **否** |
| `mat_pending` | 魔盒已装、数据采集中 | 「网关在线，城池筹建中」 | **否**（或受限预览，二期定） |
| `certified` | 魔盒 hardware 实证 / 封签完成 | 正式碳城池、验真码、官方因子 | **是** |

**禁止**：将 `evidence_building` 数据当作 `certified` 用于 Pull 或申报。

## 7. 下游可见性：两层并存 · 场景拆分

### 7.1 原则

- **场景 A（原厂自证状态）**：可实名展示（企业主动公开）。
- **场景 B（下游消费账本身份）**：默认匿名；仅 `opt-in` 后实名。
- 匿名用于反竞争保护；实名用于已授权协同动作与追责链路。

### 7.2 层 1 · 绑定链（精准）

-  audience：已与该原厂建立 supply binding 的下游 SME
- 展示：企业全名、CL-GTCID、碳强度摘要、城池态、共振阈值进度（若有）
- 动作：催办、穿透填报、共振 +1、参与阈值触发

### 7.3 层 2 · 行业共振大盘（宏观 + 实名榜）

- audience：同行业 SME / 链主
- 展示：诉求总量、税损敞口叙事、**实名**「实证中 / 共振触发中」原厂列表
- 动作：行业压力、发起绑定、指向共振阈值触发面板

### 7.4 公开 vs 私域（数据字段）

| 进 Hub 展示（原厂自证可实名） | 永不出域 |
|------------------------|----------|
| 企业名、CL 主体、行业 | 工序能耗绝对值 |
| CL-GTCID 权证号 | 煤气/焦比等明细 |
| 碳强度摘要 | LOCAL_VAULT / 精算芯 vault 明细 |
| 城池态、matBoxLocked | 魔盒采集原始波形（仅封签状态摘要） |

消费账本实名规则：

- 原厂可见真实消费主体（在其授权边界内）
- 对外展示默认匿名，仅下游主动 `opt-in` 后实名

## 8. 对话首页粗测（T0 · 共用顶口）

| 层级 | 产品 | 作用 |
|------|------|------|
| **T0** | `index.html` Widget 粗测 | 欧盟默认库、税损恐惧、进对话 / Paywall |
| **T1** | 精算芯 | 真实复盘、CL-GTCID；**不可**申报/公证/Pull |
| **T2** | 魔盒待建 | 待建城池 |
| **T3** | 魔盒封签完成 | 正式城池 |

话术纪律：粗测 **不** 暗示「算了就能报」；精算 **不** 暗示「算了就能 Pull」；仅 **T3** 宣传申报/公证/Pull。

**引导链（目标）**：

```
首页粗测 → 建档 / Paywall → 全域 Hub
  → 缺上游因子 → 绑定链共振 / 阈值触发
  → 深链「产业链主权实证中心」（精算芯）
  → 实证中 Hub 展示 → 魔盒 → 正式城池
```

## 9. Hub 工业原厂模块定位（修订）

### 9.1 Hub 做什么

- 城池 **三态** 展示与空态引导
- **绑定链** + **行业共振大盘**（两层）
- **共振阈值触发** 进度与达标通知
- 消费台账、批次核验、CBAM Pull（**仅 certified**）
- 深链精算芯（实证中心）、引导魔盒

### 9.2 Hub 不做什么

- **不** 运行 13 行业工序精算（不扩 INDS / BLUEPRINT 为第二算力）
- **不** 存储工序级绝对能耗
- **不** 将精算芯 HTML 整站嵌入为算力本体
- **不** 在代码层写死 `if (!mat) hub_down()`（用 tier + 城池态表达权限）

### 9.3 与现网模块关系（迁移方向）

| 现模块 | 路由 | 方向 |
|--------|------|------|
| `HengAI_工业原厂精算.html` | `factor-auth` | 收敛为 **城池台 + 引导 + 读 sync**；停止扩展 INDS |
| `HengAI_HeavyIndustry_Suite.html` | `origin-audit` | 保留 **可视化壳**（大盘、榜、动效）；不算力真理 |
| 精算芯 Universal Core | 官网 / :8001 | **唯一** 软件精算与 CL-GTCID 产出 |

## 10. Sync 契约（Hub 接收 · 讨论稿）

精算芯 / 魔盒 → Hub **`POST /api/v1/hub/overview/sync`**（待实现）最小字段集扩展：

### 10.1 通用

```json
{
  "syncTier": "SOFTWARE_EVIDENCE | MAT_PENDING | CERTIFIED",
  "source": "hengai_universal_core | cl_mat",
  "holder": "企业实名",
  "productionEntity": "统一社会信用代码或 CL 主体 ID",
  "industryId": "steel",
  "certificateId": "CL-GTCID-…",
  "carbonIntensity": 0,
  "cityState": "evidence_building | mat_pending | certified",
  "fundingMode": "self_paid | resonance_triggered",
  "qualityTag": {
    "matBoxLocked": false,
    "maturityTier": "evidence_in_progress | mat_pending | certified",
    "provenanceGrade": "software | hardware_sealed",
    "activeJurisdiction": "cbam"
  }
}
```

### 10.2 共振阈值触发（A2 · 可选）

```json
{
  "resonanceTarget": 0,
  "resonanceCount": 0,
  "participantCount": 0
}
```

### 10.3 Hub 处理规则

| syncTier | verified-factor-pool Pull | 绑定链展示 | 行业大盘 |
|----------|---------------------------|------------|----------|
| `SOFTWARE_EVIDENCE` | 否 | 是 · 实证中 | 是 · 实名榜 |
| `MAT_PENDING` | 否 | 是 · 待建 | 是 |
| `CERTIFIED` | 是 | 是 · 正式城池 | 是 |

L0/L1 与衔接说明字段 **可映射** 到上表；文档层 **单独标明** `SOFTWARE_EVIDENCE` ≠ 正式 L1 Pull 资格。
