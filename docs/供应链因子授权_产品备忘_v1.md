# 供应链 · 撤回因子授权 · 产品备忘 v1

> **状态**：**Hub 部分已实现**（2026-06-23 · 供应链协同批次）  
> **来源**：2026-06 对话推演 · 原厂（如江苏鼎胜新材）对沉默供应商的协同策略

---

## 1. 业务机制

| 角色 | 行为 |
|------|------|
| **原厂（上游）** | 在 `[供应链协同/邀请]` 对任意**沉默供应商**执行 **「撤回因子授权」** |
| **供应商（下游）** | 填报时若可引用原厂 **碳城池因子**（如上游铝锭碳强度），结果优于欧盟默认值；撤回后填报界面**仅余欧盟默认值** |
| **系统通知（供应商侧）** | 「因未及时提交自身数据，原厂因子已不可用；填报前需向 **[原厂企业名]** 提交申请，审批后解锁。」 |

**产品目标**：将供应商从 **被动配合** 转为 **主动申请**（需原厂审批才能重新引用因子）。

---

## 2. 操作路径（话术对齐全域中心架构）

```
[全域中心] → 左侧 [供应链协同/邀请] → 目标供应商 → 「因子配置」→ 「撤回授权」
```

AI 引导话术须使用 `[全域中心]` / `[供应链协同]`，**禁止**「右侧面板」等虚构方位。

## 3. 批量规则变更函（H 页闭环 · 2026-06 增补）

> **产品隐喻**：像给老板发报告一样——**发送、追踪、反馈** 全在同一 Hub 页闭环，不跳出 `[供应链协同/邀请]`。

### 3.1 操作路径

```
[全域中心] → [供应链协同/邀请]
  → 勾选已标记「🔒 需申请因子」的供应商（示例：30 家）
  → 顶部「批量发送规则变更函」
```

### 3.2 函件内容（系统生成 · 原厂抬头）

- **发文单位**：`{原厂企业名}-供应链管理部`（例：江苏鼎胜新材-供应链管理部）
- **事由**：因年度合规审计升级，原厂碳因子已撤回授权
- **要求**：如需继续使用我方因子填报 CBAM，须在 **5 个工作日**内提交完整原始数据申请
- **附件**：申请入口及操作步骤（链至供应商侧填报/申请页）

### 3.3 下发与追踪（同页）

| 模块 | 能力 |
|------|------|
| **历史下发记录** | 按供应商列出：发送时间、送达状态、**已读/未读** |
| **批量选择** | 支持按标签「需申请因子」一键全选 |
| **状态回写** | 供应商打开函件 → 已读；提交申请 → 待审批 |

### 3.4 API / 流水（Hub 已实现）

- `POST /api/v1/hub/supply/factor-auth/revoke` — 原厂撤回 ✅
- `POST /api/v1/hub/supply/factor-auth/apply` — 下游申请解锁 ✅
- `POST /api/v1/hub/supply/factor-auth/approve` — 原厂审批 ✅
- `POST /api/v1/hub/supply/factor-rule-letters/batch` — 批量函件 ✅
- `GET  /api/v1/hub/supply/factor-rule-letters/history` — 历史记录 ✅
- `GET  /api/v1/hub/supply-binding/mine` — 下游状态 + 通知 ✅
- `Action_Ledger`：`REVOKE_FACTOR_AUTH` / `REQUEST_FACTOR_AUTH` / `APPROVE_FACTOR_AUTH` / `REJECT_FACTOR_AUTH` ✅
- 待补：`FACTOR_RULE_LETTER_READ`（函件已读回写）
