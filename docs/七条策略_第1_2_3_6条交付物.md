## 第一条：重点用能单位名录 → 销售线索方法论

### 1.1 核实结论（已verified，非推测）

经核实，目标企业名录**不是一份全国统一数据库**，是分散在两条渠道、按省/按年度公示的：

| 渠道 | 发布层级 | 已验证的真实样本 | 对Co2Lion的价值 |
|------|---------|----------------|----------------|
| **渠道A：省级"重点用能单位名单"** | 省发展改革委，逐省公示 | 《关于调整河南省重点用能单位名单的公示（2025版）》，河南省发展和改革委员会，2025年9月29日发布 | 覆盖面最广，但企业质量参差，需要二次筛选 |
| **渠道B：重点行业能效"领跑者"企业名单** | 工信部+国家发改委+市场监管总局**联合发布（国家级）**，同时各省工信厅也单独公示省级名单 | 《2025年度重点行业能效"领跑者"企业名单》，工信部，2025年12月23日发布；福建省工信厅2025年12月11日公示省级名单（金纶高纤等3家） | **更优质的线索源，见1.2** |

### 1.2 为什么渠道B是更精准的线索（这是本次核实最有价值的发现）

"领跑者"企业的官方申报条件第5项明确写着：

> **"建立节能目标责任制和节能奖惩制度，并根据国家标准及相关要求建设能耗在线监测系统、能源管控中心或能碳管理中心"**

这意味着登上这份名单的企业，**已经满足NHJC路径Tier 0的前置条件**——它们不是"可能有"合规端设备，是"申报材料里必须证明自己有"。同时，这份名单聚焦的行业是：

> 石化化工、钢铁、有色金属、建材、轻工、纺织——**与Co2Lion的8大重工业蓝图高度重合**

这是一份"already pre-qualified"的名单：行业对、有监测系统、且因为入选"领跑者"通常意味着企业规模较大、管理较规范、更可能有出口业务（更可能真实承受CBAM压力）。

### 1.3 获取方法（具体操作步骤）

```
第一步：国家级"领跑者"名单
  → 搜索关键词："工业和信息化部 + [当年年份] + 重点行业能效领跑者企业名单"
  → 来源：工信部官网（miit.gov.cn）及"工业节能与绿色发展管理平台"（green.miit.gov.cn）

第二步：省级名单补充（逐省搜索，建议先做CBAM受影响产业集中的省份）
  → 优先省份：河北/山东/江苏/广东/河南/福建/江西（钢铁/铝业/水泥/陶瓷产业带集中区）
  → 搜索关键词："[省份] + 工业和信息化厅/发展改革委 + 重点用能单位名单/领跑者名单"

第三步：与海关出口数据交叉
  → 名单上的企业名称，逐一核对其是否有对欧盟出口记录
  → 可用：企查查/天眼查类工商数据平台的"进出口"标签初筛，或直接联系熟悉的报关行/货代获取粗略出口画像
  → 此步骤过滤掉"有监测系统但不做出口、与CBAM无关"的企业

第四步：形成内部线索评分表（模板见下）
```

### 1.4 线索评分模板

| 字段 | 说明 |
|------|------|
| 企业名称 | — |

## 第二条：历史数据回填 → 产品话术 + 技术规格

### 2.1 销售话术（可直接使用）

**核心话术（针对已确认走渠道B的企业）：**

> "您不需要从今天才开始攒碳数据。按国家《重点用能单位能耗在线监测系统技术规范》要求，您现有的能耗监测系统依法必须本地留存6个月以上历史数据——这笔家底已经存在了，接入CL-MAT/精算芯的当天，就能一次性把这6个月的数据点亮，不用从零开始。"

**针对犹豫"要不要装"的客户，补充话术：**

> "您今天不装，这6个月的历史数据每天都在过期——监测系统的留存窗口是滚动的，越晚接入，能回填的历史窗口越短。"

**这句话本身带有一个真实的紧迫感，建议作为CTA文案使用，不需要夸大，事实本身已经有足够的说服力。**

### 2.2 回填功能技术规格（对应Cursor开发）

```
功能名称：历史数据一次性回填导入
触发场景：工厂完成Tier 0路径接入（NHJC本地接口只读权限确认后）的首次同步

数据流：
  CL-MAT/精算芯 Tier0读取器
    → 读取端设备本地留存的历史分钟包（最长6个月窗口）
    → 按 payload-v1.schema.json 的 minutePacket 结构逐条归一化
    → 批量生成历史 envelope+body（每条历史数据的 capturedAt use 其原始采集时间，不是回填发生的时间）
    → 批量推送 /api/v1/mat/push（建议增加批量端点 /api/v1/mat/push-batch，避免6个月数据逐条HTTP往返）

写入 history[] 时的特殊标记：
  追加字段 "backfilled": true，区分"实时采集的数据"和"回填导入的历史数据"，
  避免历程时间线上出现"过去6个月每分钟都有一条实时数据"这种不真实的展示效果——
  回填的数据应该用一种聚合/压缩的方式呈现，比如在历程时间线上显示为
  "已回填6个月历史数据（2026-01至2026-06，共XX,XXX条分钟包）"这一条汇总记录，
  而不是把6个月的每一分钟都铺开展示。
```

## 第三条：Form C SDK合作对象 — BD优先级简报

### 3.1 目标排序（已确认市场地位，非猜测）

| 优先级 | 企业 | 理由 | 建议接触方式 |
|--------|------|------|-------------|
| **P0** | 许继电气 / 康派智能 | 同一体系（康派即原许继信息智慧能源事业部），同时覆盖电表硬件+能耗监测系统软件两层，且已有ISCCC认证产品"T@Energy-AIO"，钢铁/化工案例已验证 | 优先约谈康派智能产品负责人，切入点：CL-MAT trust-root SDK授权，帮助其产品额外获得"碳数据密码学确权"能力，作为其现有产品的增值模块，不是替代 |
| **P1** | 安科瑞 | 市场调研中案例最多、覆盖钢铁/化工/有色金属最广，且公开发布完整产品手册和Modbus协议文档，透明度高，技术对接成本最低 | 优先从其"节能控制事业部"切入，该部门面向工业客户，最贴近Co2Lion需求 |
| **P2** | 浪潮 | 有专门的重点用能单位解决方案产品线，国企/大型企业渗透率高 | 接触方式建议走更正式的渠道（如政企合作部门），周期较长 |
| **P2** | 群智合信息科技 | 多源采集集成能力强（PLC/DCS/SIS/MES/ERP），适合复杂大型工厂场景 | 技术对接优先，商务接触可稍后 |

### 3.2 谈判要点（给到BD团队的具体话术框架）

```
不要谈："我们想卖给你的客户一个新硬件"
要谈："我们想给你现有的产品加一层密码学确权能力，
      让你的客户多一个'数据有法律效力'的卖点，
      你不需要重新研发，只需要集成我们的trust-root SDK"

价值主张三句话：
  1. 你的客户已经在用你的系统做合规上报，我们不抢这个市场
  2. 我们做的是"在你已有数据基础上，加一层让欧盟海关认可的密码学签名"
  3. 这是你产品的增值能力，不是竞品替代
```

### 3.3 风险提示（重申之前的CTO红线）

这份简报是**市场优先级排序，不是已经达成的合作**。在任何对外物料、代码注释、产品文档里，不要把"这是我们的BD优先目标"写成"这是我们的合作伙伴"，两者是完全不同的法律和商业状态。
